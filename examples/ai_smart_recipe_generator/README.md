# AI Smart Recipe Generator

An AI-powered recipe generator, meal planning, and nutrition analysis web application built with the OpenViper framework. This project showcases how to use OpenViper's pluggable AI Provider Registry to interact with Gemini to create tailored recipes and detailed nutritional breakdowns.

## Features

- **Recipe Generation:** Generate custom recipes based on available ingredients, cuisine preferences, dietary restrictions, and cooking time.
- **Nutrition Analysis:** Fetch detailed macronutrient and micronutrient breakdowns generated by AI.
- **Meal Planning:** Organize generated recipes into daily or weekly meal plans.
- **AI Registry Integration:** Leverages OpenViper's core AI interfaces for consistent, async-first LLM querying.
- **Modern UI:** Uses OpenViper templates and static assets to deliver a responsive frontend experience.

## Running with Docker Compose

This project contains a Docker Compose environment that provisions the OpenViper application, a PostgreSQL database, and an Nginx reverse proxy.

### 1. Set your Gemini API Key

Create a `.env` file in the root of the project (`examples/ai_smart_recipe_generator/.env`) and add your Gemini API key:

```env
GEMINI_API_KEY=your_gemini_api_key_here
```

### 2. Start the Services

Run the following command to build and start the containers:

```bash
docker compose up --build
```

### 3. Access the Application

Once the containers are up and healthy, you can access the application at:
- **Web Interface:** `http://localhost:8080` (routes through Nginx proxy)
- **Admin:** `http://localhost:8080/admin`
- **Swagger API Docs:** `http://localhost:8080/open-api/docs`
